# Skill: Customer-Specific SKILL.md and DESIGN.md

`SKILL.md` defines content truth, reusable components, page goals, interactions, data and integration boundaries, accessibility and performance rules, asset rights, and acceptance tests.

`DESIGN.md` defines the approved visual direction, palette, typography, spacing, layout grammar, imagery, motion, desktop and mobile behavior, scene order, poster system, and reference links.

Both files must cite the customer dossier, source URLs, customer-owned assets, and chosen design option. They may not contain secrets, invented claims, unlicensed assets, or unapproved scope. The Website Agent may not begin implementation until both files exist and the applicable customer approval is durable.
