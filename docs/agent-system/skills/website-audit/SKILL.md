# Skill: Website Quality and Conversion Audit

Review accessibility, usability, content, conversion, SEO, performance, and technical quality using public pages and customer-provided evidence. For every item record the page or route, evidence, affected user or business task, severity, confidence, proposed fix, expected benefit, effort, source, and safe retest method.

Use careful labels such as candidate, verified, rejected, or resolved. Do not submit forms, create accounts, enter checkout, access private routes, bypass controls, or run active security tests without current written authorization. Never turn passive observations into breach, vulnerability, compliance, or legal claims without appropriate evidence and human review.
