# Skill: Visual, Functional, Accessibility, Security, and Performance QA

Serve the application over HTTP and test every route and meaningful state at desktop, tablet, and mobile sizes. Compare screenshots to the approved references and map each reference to the implemented page or section.

Test keyboard navigation, visible focus, headings and landmarks, form validation and errors, direct routes, authentication boundaries, network failures, responsive overflow, zoom and reflow, reduced motion, touch release, media lifecycle and memory, console/page/media errors, MIME types, security headers, dependency audit, and performance budgets.

Use synthetic data. Do not submit production forms or payments unless the test is explicitly authorized and isolated. Repeat the build-test-fix loop until every required acceptance criterion passes or a limitation is explicitly documented and approved.
